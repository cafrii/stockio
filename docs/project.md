## 프로젝트 개요 및 목표

주식 시세 정보 중계 서비스를 개발한다.
프로젝트의 상세 내용은 PRD 문서인 docs/stockio_prd.md 를 참고한다.

과거에 진행했던 PoC 작업의 기록이 남아 있다. docs/poc_summary.md 를 참고한다.

## 진행 상황 추적

목표 수행을 위해 PRD 에서 정한 개략적인 마일스톤은 milestone.md 로 분리하여 관리한다.
각 마일스톤 하부에 보다 상세한 업무를 TODO 목록으로 작성한다.
각 마일스톤 (또는 phase) 별로 산출물을 docs 폴더 아래에 정리하여 저장한다.
그리고 docs/status.md 에는 현재 까지의 진행 상황을 간략 요약한다.
status.md 에는 다음에 할 일이 간단하게 기술되어 있어야 한다.

만약 이 단계에서 작성된 단위시험/E2E 시험이 있다면 잠시 중단하고 사용자의 리뷰 및 피드백을 기다린다.

## 주요 결과물
- PRD 에 맞춰 수립된 각 단계에 따라 작업을 수행한 후 결과물을 생성한다.
  - 계획 단계: 계획표, TODO, 마일스톤, 일정.
  - 구현 단계: 코드
  - 테스트 단계: 테스트 리포트
  - 시장조사, 트렌드분석 등: 보고서
- 문서에 해당되는 결과물은 docs/ 폴더 아래에 생성한다. 다음은 예시이다.
  - milestone.md
  - architecture.md
- 중복되는 내용이 이미 다른 문서에 기술되어 있다면, 굳이 반복 작성 할 필요 없이 "xxx.md 문서를 참고한다" 정도로 언급만 해도 된다.

## 중요 지침
- 대화 컨텍스트가 리셋이 되더라도 LLM이 마지막 수행한 작업부터 resume 할 수 있어야 한다. 이를 위해 각 TODO 항목 (또는 마일스톤, phase) 완료 후에는 docs/status.md 를 업데이트 한다.
- status.md 에는 새로운 대화 창 컨텍스트에서 다시 시작되더라도 직전 작업을 끊김 없이 재개 할수 있을 만한 충분한 정보가 포함되어 있어야 한다.
- 마일스톤이 하나 끝난 후에는 사용자의 리뷰 및 검증 테스트를 수행하기 위해 일단 중단한다. 사용자가 테스트를 수행하는 방법을 간단하게 설명한 후 feedback을 기다린다.
- status.md 의 내용은 다른 문서의 내용을 참조할 수 있는 경우 간략하게 기술할 수 있다.
  - 예시: "2단계 완료. 결과물은 phase2_report.md 로 작성하였음."

## 기타 지침
- 마일스톤 TODO 항목 하나가 완료되면 TODO 체크하고, status 업데이트 하고, 사용자 리뷰/피드백을 대기한다.

## 참고할 사항
- 독창적이고 새로운 아이디어를 적극 반영하라.
- 내가 속한 회사는 개발 리소스가 넉넉하지 않은 스타트업 이지만, 검토 할 때 너무 현실 상황에 얽매이지는 않도록 하라. 목표는 가급적 원대하고 크게 설정한다.
- 틈새 시장 공략을 적극적으로 모색하라.

## 기타
- 대화는 한글을 사용하라. 비 영문의 경우 UTF-8 인코딩을 사용하라.
